形状(Shape)：
	方块最基本的形状，默认为完整方块的形状(Full Block)
	视线指上去的时候显示的方框，部分方块没有轮廓箱，Wiki或玩家常用这种说法。
	但是大部分方块都有属于自己的特殊形状

轮廓(Outline)：
	视线指上去的时候显示的方框，部分方块没有轮廓箱。
	但是没有在代码里找到被wiki成为轮廓形状的代码部分，个人猜测所谓轮廓形状就是	形状(Shape)

互动形状(InteractionShape)：
	取代轮廓形状，用来决定交互方向的形状。
	几乎所有的方块都为空，只有四个方块有互动形状：脚手架、炼药锅、粪桶、漏斗。

碰撞形状(CollisionShape)：
	决定实体与方块碰撞相关的形状。
	默认为方块的形状(Shape)

实体触发方块的碰撞形状(EntityInsideCollisionShape)：
	触发某种方块的“entityInside”方法的形状。此形状不是碰撞箱，只是用来触发方块的一些行为的。
	默认为完整方块的形状(Full Block)，有以下几个特例：
		末地传送门：(0, 6, 0; 16, 12, 16)
		岩浆锅：Shapes.or(AbstractCauldronBlock.SHAPE, (2, 4, 2; 14, 15, 14)) 炼药锅形状 + 内容物
		细雪锅：Shapes.or(AbstractCauldronBlock.SHAPE, (2, 4, 2; 14, 3l+6, 14)) 炼药锅形状 + 内容物
		细雪方块：
			如果：fallDistance > 2.5，那么：(0, 0, 0; 1, 0.9, 1) 
			如果: 实体是下落方块 or (实体能在细雪上行走(皮革鞋子或某些生物) and 实体底部高度>方块顶部)，那么：完整方块
			否则：空形状
		绊线：连接的线(0, 1, 0; 16, 2.5, 16)，未连接的线：半砖形状


方块支撑形状(BlockSupportShape)：
	默认为方块的碰撞箱(CollisionShape)，有以下特例：
		天花板悬挂告示牌：(a, 0, a; b, 10, b)；受到方向影响 a,b = 8±7,8±1
		侧墙悬挂告示牌：(a, 0, a; b, 10, b)；受到方向影响 a,b = 8±7,8±1
		紫颂花：box(1, 0, 1; 15, 15, 15)
		雪片：box(0, 0, 0; 16, (层数-1)*2, 16)
		栅栏门：关门：碰撞箱，开门：空
		树叶：空形状
		潜影盒：关闭：完整方块，打开：向上的盒子(其余方块会按此旋转)：box(0, 0, 0; 16, 1, 16)
		泥巴：完整方块顶部凹下去两个像素
		灵魂沙：完整方块顶部凹下去两个像素


遮挡形状(OcclusionShape)：
	用于光照传播的计算
	默认为方块的轮廓箱，特例有：
		空：苔藓地毯、滴水石锥、头颅类、铜傀儡
		碰撞箱：栅栏类、栅栏门类、讲台、幽匿尖啸体
		

视觉遮挡形状(VisualShape)：
	能否穿过方块看到自己——也就是判断第三人称视角的相机位置。
	默认为方块的碰撞箱(CollisionShape)，特例有：
		形状(Shape)：栅栏类、雪片
		完整方块(Full Block)：泥巴、灵魂沙
		空(Empty Block)：铁栅栏、玻璃板、细雪、玻璃类(玻璃、染色玻璃、遮光玻璃)

渲染形状(getRenderShape)：
	用途未知
	默认为model，以下几个为invisable：
		空气、屏障、结构空位、光源方块
		液体、气泡柱、移动中的方块
		末地门、末地折跃门
