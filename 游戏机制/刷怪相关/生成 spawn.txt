net.minecraft.world.entity.SpawnPlacementTypes
	SpawnPlacementTypes：
		NO_RESTRICTIONS (无限制) = 无条件可以
		IN_WATER (水中) = 世界边境之内 && 此方块含水 && 上方不是红石导体
		IN_LAVA (岩浆中) = 世界边境之内 && 此方块含岩浆
		ON_GROUND (在地面) = 世界边境之内 
			&& 下方方块允许生成此生物(isValidSpawn)
			&& 此方块和上方方块两个位置都是合法的生成空位(isValidEmptySpawnBlock)
	
	其中 ON_GROUND (在地面) 两个判断比较复杂：
	
		isValidSpawn (生物能否在这上面生成)：
			通用判定：上表面“方块支撑形状”完整 && 发光亮度 < 14
			特判(无视通用判定)：
				never：
					无法破坏方块：基岩、屏障、
					玻璃类：玻璃、染色玻璃、铜网、
					装饰类：活板门、紫颂花、脚手架(1.19.3+)
				always：
					两个上表面碰撞箱不完整的：灵魂沙、泥巴、
					三个光源方块：雕刻南瓜、南瓜灯、红石灯
				
				岩浆块：entityType.fireImmune()
				树叶：豹猫和鹦鹉
				冰、霜冰：北极熊
		
		isValidEmptySpawnBlock：(生物能否在内部生成)
			碰撞箱不能为完整(!isCollisionShapeFullBlock)
			&& 不能是红石信号源(isSignalSource())
			&& 不能含有液体(fluidState.isEmpty())
			&& 不是含有阻止怪物在内部生成的标签(!BlockTags.PREVENT_MOB_SPAWNING_INSIDE)(目前只包含四种铁轨)
			&& 不是对于这个生物危险的方块(!entityType.isBlockDangerous)
		
			其中：isBlockDangerous：(检查在这个方块里是否对于这个生物危险)
				某些生物免疫某些危险方块的伤害(.fireImmune和.immuneTo())，
				如果不是以下危险方块，什么生物都返回false(不危险)
				如果是某个危险方块，此生物又是对应免疫的生物就返回false(不危险)，不是免疫的生物就返回true(危险)，
				免疫组合有：
					1.燃烧的方块(火、岩浆、岩浆块、点燃的营火、岩浆锅)：
						药水效果云、末影水晶、TNT
						恼鬼、坚守者、末影龙、潜影贝、凋零
						烈焰人、恶魂、岩浆怪、炽足兽、凋灵骷髅、僵尸疣猪兽、僵尸猪灵
					
					2.甜浆果：狐狸
					3.细雪：雪傀儡、北极熊、流浪者
					4.凋灵玫瑰：凋灵、凋灵骷髅
					5.仙人掌：(无)
